AUnit Coverage

What is Coverage in AUnit?

When a test case is execute: What part of the model can be covered? What should that coverage criteria look like?
In this section we will outline:
      a. What are the coverage criteria in Alloy?
      b. How a test case's execution covers these criteria.
      c. How coverage is displayed in the Alloy Analyzer.

Coverage Criteria

We consider criteria over 4 main structures within Alloy models: signatures. relations, expression and formulas.

Signature s Criteria:
      1. |s| = 0 (empty set)
      2. |s| = 1 (singleton set)
      3. |s| > 1 (no empty, no singleton set)

Relation r Criteria:
      1. |r| = 0 (empty set)
      2. |r| = 1 (singleton set)
      3. |r| > 1 (no empty, no singleton set)

Expression e Criteria:
      1. |e| = 0 (empty set)
      2. |e| = 1 (singleton set)
      3. |e| > 1 (no empty, no singleton set)

Formula f Criteria:
      1. f = true
      2. f = false
Additionally, for quantified formulas ("q v : d | b") where q = quantifier, v = variable, d = domain, b = subformula:
      3. |d| = 0
      4. |d| = 1, b = true
      5. |d| = 1, b = false
      6. |d| > 1, b = true
      7. |d| > 1, b = false
      8. |d| > 1, b = true & false

How a test case's execution covers these criteria

When a test case is execute, coverage is calculated for:
      a. All signatures in the model
      b. All relation in the model
      c. All expression and formulas in the facts
      d. All expression and formulas in any predicates invoked in the test case's command.

How Coverage is Displayed

Coverage is displayed in two ways:
      a. Highlighting the model: green means covered, yellow means partially covered, and red means not covered.
      b. Coverage displayed by paragraph on the "Coverage Result" tab. This coverage is broken down my paragraph and each view can be expanded to show which specific criteria have or have not been covered.
