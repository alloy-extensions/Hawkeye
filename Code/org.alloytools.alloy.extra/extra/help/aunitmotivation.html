Motivation for AUnit

Common ad-hoc approaches to testing Alloy models include: building formulas to test the relationship between predicates and enumerating instances -- looking for either the presence or abscene of a particular instance. AUnit is intended to bring a uniform verification process to Alloy. Our valuation structure allows users to continue to test Alloy models in the same spirit as the ad-hoc approaches, but with a built in infrastructure to help the user target specfic instances or formulas.

In Alloy, the visual instances that get enumerated over are satisfying solutions found by the back-end SAT solvers for the formulas being invoked. The relation-based nature of Alloy often enables these instances to be displayed in an approachable nature; i.e. a binary relation is a graph. AUnit test cases leverage these ideas to bring testing to Alloy. Our valuations model these instances and our test case commands enable a tester to specify whether these instances should or should not have been allowed by various Alloy formulas.
